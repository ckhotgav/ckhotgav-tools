# project-init — 啟動專案 / 初始化專案

當使用者說「**啟動專案**」或「**初始化專案**」時，請執行以下流程。

---

## 步驟一：詢問專案名稱

用簡單的方式問使用者，例如：
> 「好的！我們來建立一個新專案！這個專案叫什麼名字呢？」
> （例如：「我的個人網站」、「學習 Python 筆記」）

---

## 步驟二：詢問專案類型

讓使用者選擇專案類型。用選項的方式呈現，比較好懂：

> 「這個專案是做什麼用的呢？請選一個：
>
> **1️⃣  靜態網頁** — 就是一般的網站，用 HTML / CSS / JavaScript 就可以
>
> **2️⃣  Node.js 應用** — 需要後端伺服器的網站或工具
>
> **3️⃣  Python 工具** — 用 Python 寫的自動化工具或程式
>
> **4️⃣  文件／筆記專案** — 純粹寫文件、做筆記
>
> **5️⃣  其他** — 都不是？告訴我你想做什麼，我來幫你決定」

---

## 步驟三：分析並建立專案結構

根據選擇的專案類型，自動建立對應的目錄結構。

### 靜態網頁
```
📁 專案名稱/
├── index.html        # 首頁
├── style.css         # 樣式
├── script.js         # 功能
└── assets/           # 圖片、字型等
    ├── images/
    └── fonts/
```

### Node.js 應用
```
📁 專案名稱/
├── package.json
├── src/
│   └── index.js
├── tests/
└── .gitignore
```

### Python 工具
```
📁 專案名稱/
├── main.py
├── requirements.txt
├── src/
├── tests/
└── .gitignore
```

### 文件／筆記專案
```
📁 專案名稱/
├── docs/
├── notes/
└── 專案名稱.md       # 專案說明文件
```

---

## 步驟四：自動產生專案設定檔

結構建立完成後，自動產生以下檔案，**全部用白話中文撰寫**：

### PROJECT.md — 專案基本資料
```markdown
# [專案名稱]

## 這個專案是？
（簡單說明這個專案要做什麼）

## 目前進度
- [ ] 尚未開始
- [ ] 進行中
- [ ] 已完成

## 技術
（列出這個專案用到的工具或語言）

## 備註
```

### AGENTS.md — 告訴 AI 這個專案的規則
```markdown
# [專案名稱] — AI 注意事項

## 這個專案在做什麼
（讓 AI 知道專案目的）

## 程式碼習慣
- 使用繁體中文註解
- 變數名稱用英文
- 每個檔案開頭說明用途

## 開發流程
1. 開工時讀取 PROJECT.md
2. 完成功能後更新進度
3. 收工時自動 commit
```

### .gitignore
自動對應專案類型，忽略不需要的檔案。

---

## 步驟五：初始化 Git

```powershell
git init
git add .
git commit -m "🎉 初始化 [專案名稱]"
```

---

## 步驟六：回報結果

用開心的口吻回報，例如：

> 「 ✅ **專案「我的個人網站」已建立完成！**
>
> 📂 資料夾結構已經排好了
> 📄 PROJECT.md 和 AGENTS.md 已自動產生
> 🔗 Git 版本控制已初始化
>
> 下一步建議你：
> 對我說「**開工**」，我來告訴你今天可以從哪裡開始！ 🚀」
