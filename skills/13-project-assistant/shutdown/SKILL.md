# shutdown — 收工

當使用者說「**收工**」時，請執行以下流程。

## 步驟一：詢問完成事項

用輕鬆的口吻問使用者今天完成了哪些事，例如：
> 「好的！今天你做了哪些事呢？簡單跟我說一下～」

讓使用者用簡單的中文描述即可。

## 步驟二：更新專案設定檔

將使用者的完成事項自動記錄到專案設定檔（如 `PROJECT.md`）的進度區塊中。

## 步驟三：記錄工作日誌

如果使用者有安裝 Obsidian 知識管理系統，在 Obsidian 中新增一篇工作日誌筆記：
- 標題格式：`YYYY-MM-DD 工作日誌`
- 內容包含：今日完成事項、遇到的問題、明日計劃

## 步驟四：備份到 GitHub

1. 執行 `git add .`
2. 執行 `git commit -m "收工：今日進度更新"`
3. 執行 `git push`

## 步驟五：完整性檢查

檢查以下項目：
- ✅ 所有檔案已儲存
- ✅ 程式無明顯錯誤（若可以執行檢查指令則執行）
- ✅ 已推送至 GitHub

## 步驟六：回報完成

用輕鬆的口吻回報，例如：
> 「下班打卡完成！今日進度已記錄 ✅
> 你完成了：完成關於我頁面、加上聯絡表單
> 明天見～」
