# shutdown — 收工 / 完成專案

當使用者說「**收工** / **完成專案** / **結束專案**」時，請執行以下流程。

## 步驟一：詢問完成事項

用輕鬆的口吻問使用者今天完成了哪些事，例如：
> 「好的！今天你做了哪些事呢？簡單跟我說一下～」

讓使用者用簡單的中文描述即可。

## 步驟二：更新專案設定檔

將使用者的完成事項自動記錄到專案設定檔（如 `PROJECT.md`）的進度區塊中。

## 步驟三：記錄工作日誌與開發日誌

如果使用者有安裝 Obsidian 知識管理系統：
- **工作日誌**：在 `收件匣/每日日誌存檔/` 新增或更新 `YYYY-MM-DD.md`
- **開發日誌**：詢問是否也要記錄到對應專案的開發記錄資料夾
- **Clippings 整理**：詢問是否要一併整理 Clippings 中的網頁剪輯

## 步驟三.5：檢查自動整理狀態

如果存在 Obsidian Vault 中的 `_整理記錄.md`（先從 MCP 設定檔或詢問使用者取得 vault 路徑）：
1. 讀取 `last_organize`、`enabled`、`interval_days`
2. 若 `enabled = true` 且距上次整理超過 `interval_days` 天：
   - 提醒：「⚠️ 已超過 N 天未整理知識庫，收工前要跑一次整理嗎？」

## 步驟四：備份到 GitHub

1. 執行 `git add .`
2. 執行 `git commit -m "收工：今日進度更新"`
3. 執行 `git push`

## 步驟四.五：雲端部署 (選用)

主動詢問使用者是否需要部署至雲端：
* **選項 1**：Netlify 一鍵部署 (netlify deploy --prod, 詳見技能 04)
* **選項 2**：Cloudflare 60分鐘免登入臨時部署 (npx wrangler deploy --temporary, 詳見技能 18)
* **選項 3**：Cloudflare 正常帳戶部署 (npx wrangler pages deploy, 詳見技能 18)

## 步驟五：完整性檢查

檢查以下項目：
- ✅ 所有檔案已儲存
- ✅ 程式無明顯錯誤（若可以執行檢查指令則執行）
- ✅ 已推送至 GitHub
- ✅ 已視需要部署至雲端環境

## 步驟六：回報完成

用輕鬆的口吻回報，例如：
> 「下班打卡完成！今日進度已記錄 ✅
> 你完成了：完成關於我頁面、加上聯絡表單
> 明天見～」
