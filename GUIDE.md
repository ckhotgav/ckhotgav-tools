## 🚀 啟動專案流程 (Project Initialization)
當您要從頭開始一個新的開發專案時，只需在編輯器對話框中輸入以下任一對話關鍵字：
> 💬 **「啟動專案」**、**「專案啟動」** 或 **「開啟專案」**

### 📋 詳細操作步驟與對答說明

#### 1. 觸發專案助理
在對話框中輸入上面任一個關鍵字，例如：`幫我開啟專案`、`啟動專案`、`進行專案啟動`。

#### 2. 設定專案名稱
AI 助理會詢問您：
> *「嗨！我是您的自動專案助理。請問您要建立的新專案名稱是什麼？（例如：`my-web-app`）」*
* **您的操作**：直接輸入您想建立的專案資料夾名稱，建議使用英文、數字與減號 `-`。

#### 3. 選擇專案模板類型
接著，AI 助理會詢問您：
> *「好的，專案名稱為 `[專案名稱]`。請問這是一個什麼類型的專案？請回覆對應的編號或文字：*
> * *1️⃣ **靜態網頁** — HTML / CSS / JavaScript 網頁專案。*
> * *2️⃣ **Node.js 應用** — 使用 npm 管理的 JavaScript 後端或工具專案。*
> * *3️⃣ **Python 工具** — 用於數據處理、自動化的 Python 專案。*
> * *4️⃣ **文件／筆記專案** — 純寫文件與 Markdown 筆記的專案。*
> * *5️⃣ **其他** — 使用者自設結構與語言（如 Go, Rust, Java 等，或自訂目錄結構）。」*
* **您的操作**：直接回覆 `1` ~ `5` 即可。若選擇 `5`（其他），請告訴 AI 助理您需要的程式語言或特定的目錄檔案規劃，例如：`我需要 Go 語言專案，並包含一個 cmd/main.go 檔案`。

#### 4. 自動化建立與初始化
AI 助理會開始自動執行以下任務：
1. 建立專案目錄與基本預設檔案（包含自訂「其他」結構）。
2. 自動建立專案管理核心檔案：
   * **`README.md`**：包含快速入門指引。
   * **`PROJECT.md`**：專案開發進度表，這是讓您能使用「開工/收工」流程的核心關鍵檔案。
   * **`AGENTS.md`**：為 AI 編輯器定義本專案的專屬協作習慣與開發流程規範。
3. 自動配置防禦性 `.gitignore` 檔案。
4. 自動初始化 Git 版本控制並完成首次提交。

#### 5. 完成回報
完成後，AI 助理會呈報專案結構，並提示：
> *「🎉 專案已初始化成功！且已為您完成了首次 Git 提交。您隨時可以輸入『開工』開始開發囉！🚀」*

## 🌅 開工流程 (Startup Workflow)
每天開始工作時，只需在對話框中輸入：
> **「開工」** 或 **「我來了」**

AI 助理會自動在背景執行以下步驟：
1. **環境檢查**：確認當前目錄是否有未提交的變更，避免程式碼衝突。
2. **進度掌握**：自動讀取專案底下的 `PROJECT.md` 與 `AGENTS.md`，了解前一次開發進度與目前里程碑。
3. **日記讀取**：讀取 Obsidian 每日筆記，掌握今天的工作排程與代辦事項。
4. **狀態回報**：提供一個結構清晰的摘要，並詢問你：「我們今天準備從哪裡開始？」

---

## 🌌 收工流程 (Shutdown Workflow)
結束工作時，輸入：
> **「收工」** 或 **「下班了」**

AI 助理會引導你完成以下工作：
1. **自動存檔**：確認所有編輯中的檔案已儲存。
2. **進度更新**：自動將今日完成的任務更新至 `PROJECT.md`。
3. **日誌記錄**：自動將今日工作摘要與明日待辦事項寫入 Obsidian 每日筆記。
4. **選項式同步**（**AI 會問你，得到允許才執行**）：
   * 是否要 Commit 並 Push 到 GitHub？
   * 是否要部署到 Netlify 或雲端主機？
   * 是否要把今天的記錄同步到 NotebookLM？
5. **安全掃描**：確認你沒有在程式碼中硬編碼寫入 API Key 或密碼。

## 🛡️ 安全注意事項與 .gitignore 加強防護規範
為了保護您的程式碼隱私，防止敏感資料（如 API 金鑰、資料庫密碼、私密金鑰檔案）外洩至 GitHub 公開倉庫，本工具箱制定了以下防護指引：

1. **防禦性 `.gitignore` 自動設定**：
   在您透過自動專案助理**「啟動專案」**時，AI 助理會自動在專案目錄下建立 `.gitignore` 檔案，並預設寫入嚴格的安全排除清單：
   * **環境變數檔案**：`.env`、`.env.local`、`.env.*.local` 等，防堵存放在環境變數中的金鑰外洩。
   * **憑證與金鑰檔案**：`*.pem`、`*.key`、`*.p12`、`*.pfx`、`secrets.json`、`credentials.json` 等私鑰憑證檔。
   * **系統與編輯器暫存檔**：自動忽略 `.vscode/`、`.idea/`、`.DS_Store` 等，保持 Git 倉庫乾淨且不洩漏個人編輯器習慣。
   * **大型依賴與編譯目錄**：排除 `node_modules/`、`__pycache__/`、`.venv/` 等，加速 Git 運算。

2. **嚴格禁止代碼硬編碼 (Hardcoding)**：
   * 千萬**不要**直接將任何 Supabase Key、Gemini API Key 或資料庫密碼以純文字寫在 JS、Python 或 HTML 檔案中。
   * **正確做法**：將金鑰寫入本地建立的 `.env` 檔案中（例如 `GEMINI_API_KEY=your_key`），在程式碼中透過環境變數讀取。因為 `.gitignore` 會自動忽略 `.env`，所以這樣做能確保安全。

3. **收工時的自動安全掃描**：
   * 當您對 AI 說「收工」時，助理會自動掃描您即將提交的程式碼。若發現疑似 API 密鑰的字串，會立刻發出警告並引導您將其安全地移至環境變數。

4. **破壞性操作的安全確認**：
   * AI 助理在執行任何可能導致資料丟失的操作（例如刪除檔案、清空目錄、重置 Git）前，**必須獲得您的明確確認**。
